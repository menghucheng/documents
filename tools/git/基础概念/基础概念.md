# Git 基础概念

## 1. Git 是什么

Git 是一个分布式版本控制系统，用于跟踪文件的变化，协调多人协作开发。它允许你：
- 跟踪文件的历史修改
- 恢复到之前的版本
- 分支管理，并行开发不同功能
- 多人协作，合并代码

## 2. 核心概念

### 2.1 仓库（Repository）

仓库是 Git 存储代码和历史记录的地方，分为两种类型：

- **本地仓库**：存储在你的计算机上
- **远程仓库**：存储在远程服务器上，如 GitHub、GitLab 等

### 2.2 提交（Commit）

提交是 Git 中的基本单位，代表一组文件的快照。每个提交包含：
- 提交信息：描述这次修改的内容
- 作者和时间戳
- 指向父提交的指针
- 文件的快照

### 2.3 分支（Branch）

分支是指向提交的指针，用于并行开发不同功能。默认分支通常是 `main` 或 `master`。

### 2.4 HEAD

HEAD 是一个特殊指针，指向当前所在的分支或提交。

## 3. Git 工作区域

Git 有四个主要工作区域：

### 3.1 工作目录（Working Directory）

你实际编辑文件的地方，是 Git 仓库的检出内容。

### 3.2 暂存区（Staging Area/Index）

用于暂存你想要提交的文件修改，是工作目录和本地仓库之间的缓冲区。

### 3.3 本地仓库（Local Repository）

存储所有提交历史记录，位于 `.git` 目录中。

### 3.4 远程仓库（Remote Repository）

存储在远程服务器上的仓库，用于多人协作。

## 4. 文件状态

Git 中的文件有四种状态：

### 4.1 未跟踪（Untracked）

文件存在于工作目录中，但尚未被 Git 跟踪。

### 4.2 已修改（Modified）

文件已被修改，但尚未添加到暂存区。

### 4.3 已暂存（Staged）

文件已被修改，并且已添加到暂存区，等待提交。

### 4.4 已提交（Committed）

文件已被提交到本地仓库。

## 5. Git 工作流程

1. 在工作目录中修改文件
2. 将修改的文件添加到暂存区
3. 将暂存区的文件提交到本地仓库
4. 将本地仓库的提交推送到远程仓库

## 6. 常见术语

- **克隆（Clone）**：从远程仓库复制到本地
- **拉取（Pull）**：从远程仓库获取最新代码并合并到本地
- **推送（Push）**：将本地仓库的提交推送到远程仓库
- **合并（Merge）**：将一个分支的修改合并到另一个分支
- **冲突（Conflict）**：合并时两个分支对同一文件的同一部分进行了不同修改，需要手动解决
