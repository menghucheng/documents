# Markdown 基础语法

## 1. 概述

Markdown 是一种轻量级标记语言，由 John Gruber 于 2004 年创建，旨在让人们能够使用易读易写的纯文本格式编写文档，然后转换成有效的 XHTML（或 HTML）文档。

Markdown 的主要特点：
- 简单易用，学习成本低
- 纯文本编写，兼容性强
- 可以转换为 HTML、PDF、Word 等多种格式
- 广泛应用于文档编写、博客写作、README 文件等场景

## 2. 标题

使用 `#` 符号表示标题，`#` 的数量表示标题级别，最多支持 6 级标题。

**语法示例**：

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

**显示效果**：

# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

## 3. 段落和换行

### 3.1 段落

Markdown 中的段落是由一个或多个连续的文本行组成，段落之间用一个或多个空行分隔。

**语法示例**：

```markdown
这是第一个段落。这是第一个段落的延续，
中间的换行不会产生新的段落，只会在同一行显示。

这是第二个段落，与第一个段落之间有一个空行。
```

### 3.2 换行

如果需要在段落内换行，可以在该行末尾添加两个或多个空格，然后按回车键。

**语法示例**：

```markdown
这是第一行，末尾添加两个空格  
这是第二行，与第一行在同一段落内。
```

**显示效果**：

这是第一行，末尾添加两个空格  
这是第二行，与第一行在同一段落内。

## 4. 强调

### 4.1 斜体

使用 `*` 或 `_` 包裹文本表示斜体。

**语法示例**：

```markdown
这是 *斜体文本*，这也是 _斜体文本_。
```

**显示效果**：

这是 *斜体文本*，这也是 _斜体文本_。

### 4.2 粗体

使用 `**` 或 `__` 包裹文本表示粗体。

**语法示例**：

```markdown
这是 **粗体文本**，这也是 __粗体文本__。
```

**显示效果**：

这是 **粗体文本**，这也是 __粗体文本__。

### 4.3 粗斜体

使用 `***` 或 `___` 包裹文本表示粗斜体。

**语法示例**：
```markdown
这是 ***粗斜体文本***，这也是 ___粗斜体文本___。
```

**显示效果**：

这是 ***粗斜体文本***，这也是 ___粗斜体文本___。

## 5. 列表

### 5.1 无序列表

使用 `-`、`*` 或 `+` 作为列表标记，后跟一个空格，然后是列表项内容。

**语法示例**：

```markdown
- 无序列表项 1
- 无序列表项 2
  - 嵌套无序列表项 2.1
  - 嵌套无序列表项 2.2
* 无序列表项 3
+ 无序列表项 4
```

**显示效果**：

- 无序列表项 1
- 无序列表项 2
  - 嵌套无序列表项 2.1
  - 嵌套无序列表项 2.2
* 无序列表项 3
+ 无序列表项 4

### 5.2 有序列表

使用数字加 `.` 作为列表标记，后跟一个空格，然后是列表项内容。

**语法示例**：

```markdown
1. 有序列表项 1
2. 有序列表项 2
   1. 嵌套有序列表项 2.1
   2. 嵌套有序列表项 2.2
3. 有序列表项 3
```

**显示效果**：

1. 有序列表项 1
2. 有序列表项 2
   1. 嵌套有序列表项 2.1
   2. 嵌套有序列表项 2.2
3. 有序列表项 3

### 5.3 任务列表

使用 `- [ ]` 表示未完成的任务，使用 `- [x]` 表示已完成的任务。

**语法示例**：

```markdown
- [x] 已完成的任务
- [ ] 未完成的任务
- [ ] 另一个未完成的任务
```

**显示效果**：

- [x] 已完成的任务
- [ ] 未完成的任务
- [ ] 另一个未完成的任务

## 6. 链接

### 6.1 行内链接

使用 `[链接文本](链接地址 "可选标题")` 表示行内链接。

**语法示例**：

```markdown
这是一个 [行内链接](https://www.example.com)，
这是一个带标题的 [行内链接](https://www.example.com "示例网站")。
```

**显示效果**：

这是一个 [行内链接](https://www.example.com)，
这是一个带标题的 [行内链接](https://www.example.com "示例网站")。

### 6.2 参考链接

使用 `[链接文本][链接标识符]` 定义链接引用，然后在文档的其他位置定义 `[链接标识符]: 链接地址 "可选标题"`。

**语法示例**：

```markdown
这是一个 [参考链接][1]，
这是另一个 [参考链接][example]。

[1]: https://www.example.com
[example]: https://www.example.com "示例网站"
```

**显示效果**：

这是一个 [参考链接][1]，
这是另一个 [参考链接][example]。

[1]: https://www.example.com
[example]: https://www.example.com "示例网站"

### 6.3 自动链接

使用 `<>` 包裹 URL 或邮箱地址，Markdown 会自动将其转换为链接。

**语法示例**：

```markdown
自动链接：<https://www.example.com>
自动邮箱链接：<example@example.com>
```

**显示效果**：

自动链接：<https://www.example.com>
自动邮箱链接：<example@example.com>

## 7. 图片

### 7.1 行内图片

使用 `![图片替代文本](图片地址 "可选标题")` 表示行内图片。

**语法示例**：

```markdown
![示例图片](https://via.placeholder.com/150)
![带标题的示例图片](https://via.placeholder.com/150 "示例图片标题")
```

**显示效果**：

![示例图片](https://via.placeholder.com/150)
![带标题的示例图片](https://via.placeholder.com/150 "示例图片标题")

### 7.2 参考图片

与参考链接类似，使用 `![图片替代文本][图片标识符]` 定义图片引用，然后在文档的其他位置定义 `[图片标识符]: 图片地址 "可选标题"`。

**语法示例**：

```markdown
![示例图片][img1]
![带标题的示例图片][img2]

[img1]: https://via.placeholder.com/150
[img2]: https://via.placeholder.com/150 "示例图片标题"
```

**显示效果**：

![示例图片][img1]
![带标题的示例图片][img2]

[img1]: https://via.placeholder.com/150
[img2]: https://via.placeholder.com/150 "示例图片标题"

## 8. 代码

### 8.1 行内代码

使用反引号 `` ` `` 包裹代码片段，表示行内代码。

**语法示例**：

```markdown
这是一段 `console.log("Hello World");` 行内代码。
```

**显示效果**：

这是一段 `console.log("Hello World");` 行内代码。

### 8.2 代码块

使用三个反引号 `` ``` `` 包裹代码块，并可以指定语言类型以启用语法高亮。

**语法示例**：

````markdown
```javascript
// 这是一段 JavaScript 代码
function hello() {
  console.log("Hello World");
}
hello();
```

```python
# 这是一段 Python 代码
def hello():
    print("Hello World")

hello()
```

```
// 这是一段没有指定语言的代码
function hello() {
  console.log("Hello World");
}
hello();
```
````

**显示效果**：

```javascript
// 这是一段 JavaScript 代码
function hello() {
  console.log("Hello World");
}
hello();
```

```python
# 这是一段 Python 代码
def hello():
    print("Hello World")

hello()
```

```
// 这是一段没有指定语言的代码
function hello() {
  console.log("Hello World");
}
hello();
```

## 9. 引用

使用 `>` 符号表示引用，可以嵌套多层引用。

**语法示例**：

```markdown
> 这是一段引用文本。
> 这是引用文本的第二行。

> 这是第一层引用
> > 这是第二层引用
> > > 这是第三层引用

> 引用中可以包含 **粗体**、*斜体* 等格式。
> 
> 引用中可以包含列表：
> - 列表项 1
> - 列表项 2
> 
> 引用中可以包含链接：
> [示例链接](https://www.example.com)
```

**显示效果**：

> 这是一段引用文本。
> 这是引用文本的第二行。

> 这是第一层引用
> > 这是第二层引用
> > > 这是第三层引用

> 引用中可以包含 **粗体**、*斜体* 等格式。
> 
> 引用中可以包含列表：
> - 列表项 1
> - 列表项 2
> 
> 引用中可以包含链接：
> [示例链接](https://www.example.com)

## 10. 分割线

使用三个或更多的 `-`、`*` 或 `_` 符号表示分割线，符号之间可以有空格。

**语法示例**：

```markdown
这是分割线上面的内容
---
这是分割线下面的内容

***

另一条分割线

___

第三条分割线
```

**显示效果**：

这是分割线上面的内容
---
这是分割线下面的内容

***

另一条分割线

___

第三条分割线

## 11. 表格

使用 `|` 分隔单元格，使用 `-` 分隔表头和表格内容。

**语法示例**：

```markdown
| 表头 1 | 表头 2 | 表头 3 |
|--------|--------|--------|
| 内容 1 | 内容 2 | 内容 3 |
| 内容 4 | 内容 5 | 内容 6 |

# 对齐方式
| 左对齐 | 居中对齐 | 右对齐 |
|:-------|:--------:|-------:|
| 内容 1 | 内容 2   | 内容 3 |
| 内容 4 | 内容 5   | 内容 6 |
```

**显示效果**：

| 表头 1 | 表头 2 | 表头 3 |
|--------|--------|--------|
| 内容 1 | 内容 2 | 内容 3 |
| 内容 4 | 内容 5 | 内容 6 |

# 对齐方式
| 左对齐 | 居中对齐 | 右对齐 |
|:-------|:--------:|-------:|
| 内容 1 | 内容 2   | 内容 3 |
| 内容 4 | 内容 5   | 内容 6 |

## 12. 转义字符

使用 `\` 符号转义特殊字符，使其显示为普通字符。

**语法示例**：

```markdown
\* 这不是斜体文本 \*
\# 这不是标题
\[这不是链接\](https://www.example.com)
\` 这不是行内代码 \`
```

**显示效果**：

\* 这不是斜体文本 \*
\# 这不是标题
\[这不是链接\](https://www.example.com)
\` 这不是行内代码 \`

## 13. 脚注

使用 `[^脚注标识符]` 定义脚注引用，然后在文档的其他位置定义 `[^脚注标识符]: 脚注内容`。

**语法示例**：

```markdown
这是一段带有脚注的文本[^1]，
这是另一段带有脚注的文本[^note]。

[^1]: 这是第一个脚注的内容。
[^note]: 这是第二个脚注的内容，可以包含 **粗体**、*斜体* 等格式。
```

**显示效果**：

这是一段带有脚注的文本[^1]，
这是另一段带有脚注的文本[^note]。

[^1]: 这是第一个脚注的内容。
[^note]: 这是第二个脚注的内容，可以包含 **粗体**、*斜体* 等格式。

## 14. 总结

Markdown 基础语法简单易用，涵盖了文档编写的基本需求。通过掌握这些基础语法，可以快速编写清晰、易读的 Markdown 文档。

Markdown 的优点在于其简洁性和可读性，它允许作者专注于内容创作，而不必过多关注格式。同时，Markdown 可以轻松转换为 HTML、PDF 等多种格式，适用于多种场景，如文档编写、博客写作、README 文件等。

要深入学习 Markdown，可以进一步了解其高级用法，如扩展语法、数学公式、图表等，以及各种 Markdown 编辑器和工具的使用。
