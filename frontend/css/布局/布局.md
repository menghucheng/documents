# CSS 布局

## 1. 布局基础概念

CSS布局是指在网页中安排和组织元素的方式。一个好的布局可以使网页结构清晰、美观，并且具有良好的响应式性能。

### 1.1 布局模型

CSS提供了多种布局模型，包括：

1. **正常流布局** - 默认的布局方式，元素按照HTML文档的顺序排列。
2. **浮动布局** - 使用`float`属性使元素脱离正常流，左右浮动。
3. **定位布局** - 使用`position`属性精确控制元素的位置。
4. **Flexbox布局** - CSS3引入的弹性盒模型，用于一维布局。
5. **Grid布局** - CSS3引入的网格布局，用于二维布局。
6. **表格布局** - 使用`display: table`创建表格布局。

### 1.2 盒模型与布局

CSS盒模型是布局的基础，每个元素都被视为一个矩形盒子，包含内容、内边距、边框和外边距。理解盒模型对于掌握布局至关重要。

#### 盒模型示意图

```
┌───────────────────────────────────────────────────┐
│                   Margin                         │
│  ┌─────────────────────────────────────────────┐  │
│  │                  Border                     │  │
│  │  ┌─────────────────────────────────────┐    │  │
│  │  │                 Padding             │    │  │
│  │  │  ┌─────────────────────────────┐    │    │  │
│  │  │  │        Content Area         │    │    │  │
│  │  │  │                             │    │    │  │
│  │  │  └─────────────────────────────┘    │    │  │
│  │  │                                     │    │  │
│  │  └─────────────────────────────────────┘    │  │
│  │                                            │  │
│  └─────────────────────────────────────────────┘  │
│                                                   │
└───────────────────────────────────────────────────┘
```

**盒模型各部分说明**：
- **Content Area**：实际内容区域，包含文本、图片等
- **Padding**：内容区域与边框之间的空间，透明
- **Border**：围绕内边距的边界线
- **Margin**：元素与其他元素之间的空间，透明

### 1.3 文档流

文档流是指元素在页面中的排列顺序，默认情况下，元素按照HTML文档的顺序从上到下、从左到右排列。

- **块级元素**：独占一行，宽度默认占满父元素。
- **行内元素**：不独占一行，宽度由内容决定。
- **行内块元素**：不独占一行，但可以设置宽度和高度。

## 2. 浮动布局

浮动布局是CSS早期的布局方式，用于实现文字环绕效果和多列布局。

### 2.1 float 属性

`float`属性可以使元素脱离正常流，向左或向右浮动。

```css
/* 向左浮动 */
.float-left {
  float: left;
}

/* 向右浮动 */
.float-right {
  float: right;
}

/* 清除浮动 */
.clear {
  float: none;
}
```

### 2.2 浮动布局示例

```html
<div class="container">
  <div class="sidebar">Sidebar</div>
  <div class="main-content">Main Content</div>
  <div class="footer">Footer</div>
</div>
```

```css
.sidebar {
  width: 25%;
  float: left;
  background-color: #f0f0f0;
  padding: 20px;
}

.main-content {
  width: 75%;
  float: left;
  padding: 20px;
}

.footer {
  clear: both;
  background-color: #333;
  color: white;
  padding: 20px;
}
```

### 2.3 清除浮动

浮动元素会脱离正常流，导致父元素高度塌陷。为了解决这个问题，需要清除浮动。

#### 2.3.1 使用 clear 属性

```css
.clearfix::after {
  content: "";
  display: table;
  clear: both;
}
```

#### 2.3.2 使用 overflow 属性

```css
.container {
  overflow: hidden;
  /* 或 overflow: auto; */
}
```

#### 2.3.3 使用 display: flow-root

```css
.container {
  display: flow-root;
}
```

### 2.4 浮动布局的优缺点

**优点**：
- 简单易用，兼容性好
- 适合实现文字环绕效果

**缺点**：
- 容易导致父元素高度塌陷
- 不适合复杂布局
- 难以实现垂直居中
- 响应式设计比较复杂

## 3. 定位布局

定位布局使用`position`属性精确控制元素的位置，使元素脱离正常流。

### 3.1 position 属性

`position`属性有以下几个值：

- `static`：默认值，元素按照正常流排列。
- `relative`：相对定位，元素相对于其正常位置偏移。
- `absolute`：绝对定位，元素相对于最近的已定位祖先元素偏移。
- `fixed`：固定定位，元素相对于浏览器窗口偏移。
- `sticky`：粘性定位，元素在滚动到特定位置时固定。

### 3.2 相对定位

```css
.relative {
  position: relative;
  top: 10px;
  left: 20px;
}
```

### 3.3 绝对定位

```css
.parent {
  position: relative;
}

.absolute {
  position: absolute;
  top: 0;
  right: 0;
}
```

### 3.4 固定定位

```css
.fixed {
  position: fixed;
  bottom: 0;
  left: 0;
  width: 100%;
  background-color: #333;
  color: white;
  padding: 10px;
}
```

### 3.5 粘性定位

```css
.sticky {
  position: sticky;
  top: 0;
  background-color: #f0f0f0;
  padding: 10px;
  z-index: 100;
}
```

### 3.6 z-index 属性

`z-index`属性用于控制元素的堆叠顺序，值越大，元素越靠上。

```css
.layer-1 {
  z-index: 1;
}

.layer-2 {
  z-index: 2;
}

.layer-3 {
  z-index: 3;
}
```

### 3.7 定位布局示例

```html
<div class="container">
  <div class="header">Header</div>
  <div class="sidebar">Sidebar</div>
  <div class="main-content">Main Content</div>
  <div class="footer">Footer</div>
</div>
```

```css
.container {
  position: relative;
  min-height: 100vh;
}

.header {
  height: 60px;
  background-color: #333;
  color: white;
  line-height: 60px;
  padding: 0 20px;
}

.sidebar {
  width: 200px;
  position: absolute;
  top: 60px;
  bottom: 60px;
  background-color: #f0f0f0;
  padding: 20px;
}

.main-content {
  margin-left: 240px;
  padding: 20px;
  min-height: calc(100vh - 120px);
}

.footer {
  height: 60px;
  background-color: #333;
  color: white;
  line-height: 60px;
  padding: 0 20px;
  position: absolute;
  bottom: 0;
  left: 0;
  right: 0;
}
```

## 4. Flexbox 布局

Flexbox（弹性盒模型）是CSS3引入的一维布局模型，用于实现灵活的响应式布局。

### 4.1 flex 容器

通过设置`display: flex`或`display: inline-flex`创建弹性容器。

```css
.container {
  display: flex;
}

.inline-container {
  display: inline-flex;
}
```

### 4.3 主轴和交叉轴

Flexbox布局有两个主要轴线：

- **主轴**：默认是水平方向，从左到右。
- **交叉轴**：默认是垂直方向，从上到下。

#### 主轴与交叉轴示意图

```
┌─────────────────────────────────────────────────┐
│           Main Axis (flex-direction: row)       │
│  ┌───┐ ┌───┐ ┌───┐ ┌───┐                     │
│  │ 1 │ │ 2 │ │ 3 │ │ 4 │                     │
│  └───┘ └───┘ └───┘ └───┘                     │
│        ↑ Cross Axis                           │
└─────────────────────────────────────────────────┘
```

#### 不同 flex-direction 下的轴线方向

| flex-direction | 主轴方向 | 交叉轴方向 |
|---------------|----------|------------|
| `row`         | 水平从左到右 | 垂直从上到下 |
| `row-reverse` | 水平从右到左 | 垂直从上到下 |
| `column`      | 垂直从上到下 | 水平从左到右 |
| `column-reverse` | 垂直从下到上 | 水平从左到右 |

### 4.3 容器属性

#### 4.3.1 flex-direction

控制主轴方向。

```css
/* 水平方向，从左到右（默认） */
.container {
  flex-direction: row;
}

/* 水平方向，从右到左 */
.container {
  flex-direction: row-reverse;
}

/* 垂直方向，从上到下 */
.container {
  flex-direction: column;
}

/* 垂直方向，从下到上 */
.container {
  flex-direction: column-reverse;
}
```

#### 4.3.2 flex-wrap

控制是否换行。

```css
/* 不换行（默认） */
.container {
  flex-wrap: nowrap;
}

/* 换行，第一行在上方 */
.container {
  flex-wrap: wrap;
}

/* 换行，第一行在下方 */
.container {
  flex-wrap: wrap-reverse;
}
```

#### 4.3.3 flex-flow

`flex-direction`和`flex-wrap`的简写。

```css
.container {
  flex-flow: row wrap;
}
```

#### 4.3.4 justify-content

控制主轴上的对齐方式。

#### justify-content 示意图

```
/* flex-direction: row */

flex-start:
┌─────────────────────────────────────┐
│┌───┐ ┌───┐ ┌───┐                     │
││ 1 │ │ 2 │ │ 3 │                     │
│└───┘ └───┘ └───┘                     │
└─────────────────────────────────────┘

flex-end:
┌─────────────────────────────────────┐
│                     ┌───┐ ┌───┐ ┌───┐│
│                     │ 1 │ │ 2 │ │ 3 ││
│                     └───┘ └───┘ └───┘│
└─────────────────────────────────────┘

center:
┌─────────────────────────────────────┐
│           ┌───┐ ┌───┐ ┌───┐         │
│           │ 1 │ │ 2 │ │ 3 │         │
│           └───┘ └───┘ └───┘         │
└─────────────────────────────────────┘

space-between:
┌─────────────────────────────────────┐
│┌───┐         ┌───┐         ┌───┐     │
││ 1 │         │ 2 │         │ 3 │     │
│└───┘         └───┘         └───┘     │
└─────────────────────────────────────┘

space-around:
┌─────────────────────────────────────┐
│   ┌───┐     ┌───┐     ┌───┐         │
│   │ 1 │     │ 2 │     │ 3 │         │
│   └───┘     └───┘     └───┘         │
└─────────────────────────────────────┘

space-evenly:
┌─────────────────────────────────────┐
│   ┌───┐   ┌───┐   ┌───┐             │
│   │ 1 │   │ 2 │   │ 3 │             │
│   └───┘   └───┘   └───┘             │
└─────────────────────────────────────┘
```

```css
/* 左对齐（默认） */
.container {
  justify-content: flex-start;
}

/* 右对齐 */
.container {
  justify-content: flex-end;
}

/* 居中对齐 */
.container {
  justify-content: center;
}

/* 两端对齐，项目之间的间隔相等 */
.container {
  justify-content: space-between;
}

/* 每个项目两侧的间隔相等 */
.container {
  justify-content: space-around;
}

/* 项目之间的间隔相等，两端与容器边缘的间隔也相等 */
.container {
  justify-content: space-evenly;
}
```

#### 4.3.5 align-items

控制交叉轴上的对齐方式。

```css
/* 顶部对齐（默认） */
.container {
  align-items: flex-start;
}

/* 底部对齐 */
.container {
  align-items: flex-end;
}

/* 居中对齐 */
.container {
  align-items: center;
}

/* 基线对齐 */
.container {
  align-items: baseline;
}

/* 拉伸填充（默认，当项目没有设置高度时） */
.container {
  align-items: stretch;
}
```

#### 4.3.6 align-content

控制多行项目在交叉轴上的对齐方式，只有当项目换行时才有效。

```css
/* 顶部对齐（默认） */
.container {
  align-content: flex-start;
}

/* 底部对齐 */
.container {
  align-content: flex-end;
}

/* 居中对齐 */
.container {
  align-content: center;
}

/* 两端对齐 */
.container {
  align-content: space-between;
}

/* 每个项目行两侧的间隔相等 */
.container {
  align-content: space-around;
}

/* 拉伸填充 */
.container {
  align-content: stretch;
}
```

### 4.4 项目属性

#### 4.4.1 order

控制项目的排列顺序，值越小，排列越靠前。

```css
.item {
  order: 2;
}
```

#### 4.4.2 flex-grow

控制项目的放大比例，默认值为0，即不放大。

```css
.item {
  flex-grow: 1;
}
```

#### 4.4.3 flex-shrink

控制项目的缩小比例，默认值为1，即当空间不足时，项目会缩小。

```css
.item {
  flex-shrink: 0;
}
```

#### 4.4.4 flex-basis

控制项目在主轴上的初始大小。

```css
.item {
  flex-basis: 200px;
}
```

#### 4.4.5 flex

`flex-grow`、`flex-shrink`和`flex-basis`的简写。

```css
/* 默认值 */
.item {
  flex: 0 1 auto;
}

/* 常用值，项目会放大和缩小 */
.item {
  flex: 1;
}

/* 固定大小，不放大不缩小 */
.item {
  flex: 0 0 200px;
}
```

#### 4.4.6 align-self

控制单个项目在交叉轴上的对齐方式，覆盖容器的`align-items`属性。

```css
.item {
  align-self: center;
}
```

#### 4.5.1 两列布局

```css
.container {
```

## 5. Grid 布局

Grid（网格布局）是CSS3引入的二维布局模型，用于实现复杂的响应式布局。

### 5.1 grid 容器

通过设置`display: grid`或`display: inline-grid`创建网格容器。

```css
.container {
  display: grid;
}

.inline-container {
  display: inline-grid;
}
```

### 5.2 网格线和网格轨道

- **网格线**：分隔网格的线，包括水平线和垂直线。
- **网格轨道**：两条相邻网格线之间的区域，包括行和列。

#### 网格线与网格轨道示意图

```
┌─────────┬─────────┬─────────┬─────────┐
│         │         │         │         │
│  Grid   │  Grid   │  Grid   │  Grid   │
│  Area   │  Area   │  Area   │  Area   │
│  (1,1)  │  (2,1)  │  (3,1)  │  (4,1)  │
│         │         │         │         │
├─────────┼─────────┼─────────┼─────────┤
│         │         │         │         │
│  Grid   │  Grid   │  Grid   │  Grid   │
│  Area   │  Area   │  Area   │  Area   │
│  (1,2)  │  (2,2)  │  (3,2)  │  (4,2)  │
│         │         │         │         │
└─────────┴─────────┴─────────┴─────────┘

Column Lines: 1   2   3   4   5

Row Lines:   1
             │
             2
             │
             3
```

**说明**：
- 网格线从1开始编号
- 一个3x2的网格有4条列线和3条行线
- 每个网格区域可以通过其左上角和右下角的网格线来定位，如 `grid-area: 1 / 1 / 2 / 3` 表示从行线1到行线2，列线1到列线3的区域

### 5.3 容器属性

#### 5.3.1 grid-template-columns

定义网格的列数和宽度。

```css
/* 定义3列，每列200px */
.container {
  grid-template-columns: 200px 200px 200px;
}

/* 使用fr单位，按比例分配空间 */
.container {
  grid-template-columns: 1fr 2fr 1fr;
}

/* 混合使用不同单位 */
.container {
  grid-template-columns: 200px 1fr 2fr;
}

/* 使用repeat()函数 */
.container {
  grid-template-columns: repeat(3, 1fr);
}

/* 使用auto-fill和minmax() */
.container {
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
}
```

#### 5.3.2 grid-template-rows

定义网格的行数和高度。

```css
/* 定义2行，第一行100px，第二行200px */
.container {
  grid-template-rows: 100px 200px;
}

/* 使用fr单位 */
.container {
  grid-template-rows: 1fr 2fr;
}

/* 使用repeat()函数 */
.container {
  grid-template-rows: repeat(2, 100px);
}
```

#### 5.3.3 grid-template-areas

定义网格区域，用于命名和布局。

```css
.container {
  display: grid;
  grid-template-columns: 200px 1fr;
  grid-template-rows: 60px 1fr 60px;
  grid-template-areas: 
    "header header"
    "sidebar main"
    "footer footer";
  height: 100vh;
}

.header {
  grid-area: header;
  background-color: #333;
  color: white;
}

.sidebar {
  grid-area: sidebar;
  background-color: #f0f0f0;
}

.main {
  grid-area: main;
  background-color: white;
}

.footer {
  grid-area: footer;
  background-color: #333;
  color: white;
}
```

#### 5.3.4 grid-template

`grid-template-columns`、`grid-template-rows`和`grid-template-areas`的简写。

```css
.container {
  grid-template: 
    "header header" 60px
    "sidebar main" 1fr
    "footer footer" 60px
    / 200px 1fr;
}
```

#### 5.3.5 column-gap/row-gap/gap

控制网格项目之间的间距。

```css
/* 列间距 */
.container {
  column-gap: 20px;
}

/* 行间距 */
.container {
  row-gap: 20px;
}

/* 简写 */
.container {
  gap: 20px;
}

/* 不同的行列间距 */
.container {
  gap: 10px 20px;
}
```

#### 5.3.6 justify-content

控制网格在容器中的水平对齐方式。

```css
.container {
  justify-content: center;
}
```

#### 5.3.7 align-content

控制网格在容器中的垂直对齐方式。

```css
.container {
  align-content: center;
}
```

#### 5.3.8 justify-items

控制网格项目在单元格中的水平对齐方式。

```css
.container {
  justify-items: center;
}
```

#### 5.3.9 align-items

控制网格项目在单元格中的垂直对齐方式。

```css
.container {
  align-items: center;
}
```

### 5.4 项目属性

#### 5.4.1 grid-column-start/grid-column-end

控制项目的列位置。

```css
.item {
  grid-column-start: 1;
  grid-column-end: 3;
}
```

#### 5.4.2 grid-row-start/grid-row-end

控制项目的行位置。

```css
.item {
  grid-row-start: 1;
  grid-row-end: 3;
}
```

## 7. 响应式布局

响应式布局是指网页能够根据不同屏幕尺寸自动调整布局，提供良好的用户体验。

### 7.1 媒体查询

媒体查询是实现响应式布局的核心技术，用于根据不同屏幕尺寸应用不同的样式。

```css
/* 移动设备（默认） */
.container {
  width: 100%;
}

/* 平板设备 */
@media (min-width: 768px) {
  .container {
    width: 750px;
  }
}

/* 桌面设备 */
@media (min-width: 992px) {
  .container {
    width: 970px;
  }
}

/* 大屏幕设备 */
@media (min-width: 1200px) {
  .container {
    width: 1170px;
  }
}
```

### 7.2 移动优先设计

移动优先设计是指先为移动设备设计，然后逐渐为更大的屏幕添加样式。

```css
/* 移动设备样式（默认） */
.container {
  display: flex;
  flex-direction: column;
}

/* 平板设备及以上 */
@media (min-width: 768px) {
  .container {
    flex-direction: row;
  }
}
```

### 7.3 相对单位

使用相对单位可以使元素根据屏幕尺寸自动调整大小。

- **百分比**：相对于父元素的宽度或高度。
- **rem**：相对于根元素的字体大小。
- **em**：相对于当前元素的字体大小。
- **vw/vh**：相对于视口的宽度或高度。
- **vmin/vmax**：相对于视口的最小或最大尺寸。

### 7.4 弹性图片

确保图片能够根据容器大小自动调整。

```css
img {
  max-width: 100%;
  height: auto;
}
```

## 8. 布局最佳实践

### 8.1 选择合适的布局方式

- **简单布局**：使用正常流或浮动布局。
- **一维布局**：使用Flexbox布局。
- **二维布局**：使用Grid布局。
- **表格数据**：使用HTML表格或CSS表格布局。

### 8.2 保持布局简洁

- 避免过度嵌套，减少DOM层级。
- 保持CSS规则简洁，避免复杂的选择器。
- 使用CSS变量提高可维护性。

### 8.3 考虑性能

- 避免使用复杂的布局计算。
- 减少重排和重绘。
- 使用CSS动画代替JavaScript动画。

### 8.4 测试不同设备

- 在不同尺寸的设备上测试布局。
- 使用浏览器开发者工具模拟不同设备。
- 考虑不同浏览器的兼容性。

### 8.5 遵循设计原则

- 保持视觉一致性。
- 遵循黄金比例和网格系统。
- 确保良好的可读性和可访问性。

## 10. 总结

CSS布局是前端开发的核心技能之一，掌握不同的布局方式对于创建美观、响应式的网页至关重要。从早期的浮动布局到现代的Flexbox和Grid布局，CSS布局技术不断发展，提供了更强大、更灵活的布局能力。

在实际开发中，应根据具体需求选择合适的布局方式，结合响应式设计原则，创建出适应不同设备的优秀布局。同时，应关注浏览器兼容性和性能问题，确保网页在各种环境下都能良好运行。

随着Web技术的不断发展，CSS布局也在不断演进，未来可能会出现更多新的布局方式。作为前端开发者，应保持学习的态度，不断提升自己的布局能力。
