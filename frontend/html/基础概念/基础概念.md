# HTML 基础概念

## 1. HTML 是什么

HTML（HyperText Markup Language）是用于创建网页的标准标记语言。它不是一种编程语言，而是一种标记语言，用于描述网页的结构和内容。

HTML 使用标记标签来定义网页的各个部分，如标题、段落、链接、图像等。这些标签告诉浏览器如何显示网页内容。

## 2. HTML 的发展历史

| 版本 | 发布年份 | 主要特性 |
|------|----------|----------|
| HTML | 1991 | 基础的超文本标记 |
| HTML 2.0 | 1995 | 表单、表格 |
| HTML 3.2 | 1997 | CSS 支持、数学公式 |
| HTML 4.01 | 1999 | 样式表、脚本支持 |
| XHTML 1.0 | 2000 | 基于 XML 的 HTML |
| HTML5 | 2014 | 语义化标签、多媒体支持、本地存储 |
| HTML5.1 | 2016 | 增强的表单控件、无障碍支持 |
| HTML5.2 | 2017 | 改进的安全特性、Web 组件 |

## 3. HTML 基本结构

### 3.1 基本 HTML 文档结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>文档标题</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <!-- 页面内容 -->
    <h1>这是一级标题</h1>
    <p>这是一个段落。</p>
    <script src="script.js"></script>
</body>
</html>
```

### 3.2 主要组成部分

| 部分 | 描述 |
|------|------|
| `<!DOCTYPE html>` | 文档类型声明，告诉浏览器这是 HTML5 文档 |
| `<html>` | HTML 根元素，包含整个 HTML 文档 |
| `<head>` | 头部区域，包含元数据和引用资源 |
| `<body>` | 主体区域，包含网页的可见内容 |

### 3.3 头部元素

| 元素 | 描述 |
|------|------|
| `<title>` | 设置网页标题，显示在浏览器标签栏 |
| `<meta>` | 提供元数据，如字符集、视口设置、关键词等 |
| `<link>` | 引用外部资源，如 CSS 样式表 |
| `<style>` | 定义内部 CSS 样式 |
| `<script>` | 引用或定义 JavaScript 代码 |
| `<base>` | 设置页面中所有链接的基准 URL |

**常用 meta 标签：**

```html
<!-- 字符集声明 -->
<meta charset="UTF-8">

<!-- 视口设置，用于响应式设计 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- 页面描述，用于搜索引擎 -->
<meta name="description" content="这是一个 HTML 学习页面">

<!-- 关键词，用于搜索引擎 -->
<meta name="keywords" content="HTML, 前端, 学习">

<!-- 作者信息 -->
<meta name="author" content="Author Name">

<!-- 刷新页面 -->
<meta http-equiv="refresh" content="5; url=https://www.example.com">

<!-- 禁止缓存 -->
<meta http-equiv="Cache-Control" content="no-cache, no-store, must-revalidate">
```

## 4. HTML 标签和元素

### 4.1 标签语法

HTML 标签通常成对出现，包含开始标签和结束标签：

```html
<tagname>内容</tagname>
```

有些标签是自闭合的，不需要结束标签：

```html
<tagname>
<!-- 或 -->
<tagname />
```

### 4.2 元素属性

HTML 元素可以包含属性，提供关于元素的额外信息：

```html
<tagname attribute="value">内容</tagname>
```

属性规则：
- 属性必须包含在开始标签中
- 属性名和属性值之间用等号连接
- 属性值必须用引号括起来（单引号或双引号）
- 某些属性只有属性名，没有属性值（布尔属性）

**示例：**

```html
<a href="https://www.example.com" target="_blank" title="访问示例网站">示例网站</a>
<img src="image.jpg" alt="示例图片" width="300" height="200">
<input type="text" disabled>
```

## 5. HTML 注释

HTML 注释用于在代码中添加说明，不会在浏览器中显示：

```html
<!-- 这是一个 HTML 注释 -->
<!-- 
    这是一个多行注释
    可以包含多行内容
-->
```

注释的作用：
- 解释代码功能
- 临时隐藏代码
- 帮助其他开发者理解代码

## 6. HTML 文档类型

HTML 文档类型声明（DOCTYPE）用于告诉浏览器使用哪个 HTML 版本解析文档。对于 HTML5，使用以下声明：

```html
<!DOCTYPE html>
```

DOCTYPE 声明的特点：
- 不区分大小写
- 必须放在 HTML 文档的第一行
- 不是 HTML 标签
- 没有结束标签

## 7. 字符编码

字符编码用于确定如何将字节转换为字符。在 HTML5 中，推荐使用 UTF-8 编码，它支持世界上几乎所有的字符：

```html
<meta charset="UTF-8">
```

UTF-8 编码的优势：
- 支持所有 Unicode 字符
- 向后兼容 ASCII
- 是互联网上最常用的编码
- 被所有现代浏览器支持

## 8. 视口设置

视口（viewport）是指浏览器中显示网页的区域。在移动设备上，视口通常比屏幕小，需要进行特殊设置：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

视口属性：
- `width=device-width`：设置视口宽度等于设备宽度
- `initial-scale=1.0`：设置初始缩放比例为 1.0
- `minimum-scale=0.5`：设置最小缩放比例
- `maximum-scale=2.0`：设置最大缩放比例
- `user-scalable=no`：禁止用户缩放

## 9. HTML 实体

HTML 实体用于表示特殊字符，如小于号、大于号、引号等：

| 实体名称 | 实体编号 | 显示结果 | 描述 |
|----------|----------|----------|------|
| `&lt;` | `&#60;` | < | 小于号 |
| `&gt;` | `&#62;` | > | 大于号 |
| `&amp;` | `&#38;` | & | 和号 |
| `&quot;` | `&#34;` | " | 双引号 |
| `&apos;` | `&#39;` | ' | 单引号 |
| `&nbsp;` | `&#160;` |  | 非换行空格 |
| `&copy;` | `&#169;` | © | 版权符号 |
| `&reg;` | `&#174;` | ® | 注册商标 |
| `&trade;` | `&#8482;` | ™ | 商标符号 |
| `&yen;` | `&#165;` | ¥ | 人民币符号 |

## 10. HTML 语义化

语义化 HTML 是指使用具有明确含义的标签来描述内容，而不是仅仅为了样式或布局。

**语义化的好处：**

1. **提高代码可读性**：其他开发者更容易理解代码结构
2. **有利于 SEO**：搜索引擎更容易理解网页内容
3. **改善无障碍访问**：屏幕阅读器可以更好地解析网页结构
4. **便于维护**：代码结构清晰，易于修改和扩展

**常用语义化标签：**

- `<header>`：页面或区域的头部
- `<nav>`：导航链接区域
- `<main>`：页面的主要内容
- `<article>`：独立的内容块，如博客文章、新闻报道等
- `<section>`：文档中的一个区块
- `<aside>`：侧边栏或补充内容
- `<footer>`：页面或区域的底部
- `<figure>`：独立的媒体内容，如图片、图表等
- `<figcaption>`：媒体内容的说明
- `<time>`：表示日期或时间

## 11. HTML 与 CSS、JavaScript 的关系

HTML、CSS 和 JavaScript 是前端开发的三大核心技术：

| 技术 | 作用 | 示例 |
|------|------|------|
| HTML | 结构层，定义网页的结构和内容 | `<h1>标题</h1><p>段落</p>` |
| CSS | 样式层，定义网页的样式和布局 | `h1 { color: blue; font-size: 24px; }` |
| JavaScript | 行为层，定义网页的交互和动态效果 | `document.querySelector('h1').addEventListener('click', function() { alert('Hello!'); });` |

**它们之间的关系：**

- HTML 是基础，提供网页的结构
- CSS 用于美化 HTML 结构
- JavaScript 用于增强 HTML 的交互性
- 三者协同工作，创建完整的网页

## 12. HTML 验证

为了确保 HTML 代码的正确性，可以使用 HTML 验证工具进行检查：

1. **W3C HTML 验证器**：https://validator.w3.org/
2. **Nu Html Checker**：https://validator.w3.org/nu/

验证的好处：

- 发现语法错误和潜在问题
- 提高代码质量和可维护性
- 改善浏览器兼容性

## 13. HTML 最佳实践

### 13.1 代码组织

- 使用缩进（通常是 2 或 4 个空格）提高可读性
- 保持一致的代码风格
- 将相关的元素分组
- 使用注释解释复杂代码

### 13.2 命名规范

- 使用有意义的文件名和路径
- 类名和 ID 名使用小写字母，单词之间用连字符分隔（如 `main-content`）
- 避免使用过于简单或通用的名称（如 `div1`、`container`）

### 13.3 性能优化

- 减少 HTML 文件大小
- 使用外部 CSS 和 JavaScript 文件
- 优化图像大小和格式
- 减少 HTTP 请求数量
- 使用适当的缓存策略

### 13.4 可访问性

- 使用语义化标签
- 为图像添加 `alt` 属性
- 确保表单元素有适当的标签
- 使用 ARIA 属性增强无障碍访问
- 确保文本和背景有足够的对比度

### 13.5 兼容性

- 使用标准的 HTML 语法
- 避免使用过时或实验性的标签和属性
- 测试在不同浏览器和设备上的表现
- 考虑使用 polyfill 或降级方案

## 14. 总结

HTML 是网页开发的基础，掌握 HTML 的基础概念和语法是成为前端开发者的第一步。

通过学习 HTML，你可以：
- 理解网页的基本结构
- 创建语义化的网页内容
- 为后续学习 CSS 和 JavaScript 打下基础
- 开始构建简单的网页
